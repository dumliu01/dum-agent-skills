Foreword

About this briefing.

02

2025 年底，Claude Opus 4.5 / 4.6 的发布悄悄改变了一件事 —— 代码生成的准确率从「可以辅助」变成了「可以交付」。

这不再只是一个工具升级。它意味着研发组织里的能力分布、协作模式、价值评估方式都要被重新校准。

我们整理了过去 12 个月在真实项目中累积的实践、试错和反思，希望这份简报能帮助同行更快地穿越这场转变期。它不是技术教程，而是一份关于「研发组织如何应对范式变化」的视角合集。

内容基于真实项目数据与团队访谈；所有模型评估为定性观察，不构成正式基准。

Practice I

项目规范文档 — 给 AI 一部宪法。

12 · III

目的：确保 AI 生成的代码符合项目风格，而非每次重新猜测。

必备文档清单 
project-root/
├── CLAUDE.md           # AI 编程指南
├── PROJECT.md          # 项目概览 / 术语
├── ARCHITECTURE.md     # 系统设计 / 模块
├── STYLE_GUIDE.md      # 代码风格 / 命名
├── API_STANDARDS.md    # API 规范
├── WORKFLOW.md         # 开发流程
└── .claude/settings.json
CLAUDE.md 写法 · 示例 
## 编码标准 · TypeScript
- 使用 strict: true
- 禁止使用 any
- 函数必须有 JSDoc 注释
- 模块导出必须使用 named export

## 禁止事项
× 不要修改 .env 文件
× 不要使用外部 API 密钥
× 不要修改 package.json 依赖版本

## 我的偏好
✓ 使用 TDD 方法（先写测试）
✓ 发现问题立即反馈，不要隐瞒
✓ 代码审查给出具体建议

原则：能写在文档里的不要靠口头说，能用列表的不要用段落 — AI 与人都更友好。

Practice II

项目知识文档 — 让 AI 理解「为什么」。

13 · III

规范回答「怎么做」；知识文档回答「为什么这样做」。

示例 · 决策文档 
## 为什么用 Redis 而不是内存缓存？

背景：v1.0 用内存缓存，100K 用户时崩溃
决策：引入 Redis，支持分布式部署
约束：
  - Redis 最多 5GB （成本考虑）
  - 所有 key 必须有 TTL
  - 不能缓存密码 / token 等敏感信息

涉及：src/cache/redis-client.ts
       src/middleware/cache.ts
应该记录什么
  1. 为什么选择这个架构
    替代方案与权衡
  2. 为什么有这个限制
    业务、合规、性能约束
  3. 核心业务规则是什么
    不写在代码里的领域知识
  4. 历史决策和教训
    已经失败过的方案
  5. 已知的性能瓶颈
    未优化的部分及原因
  6. 未来的扩展方向
    下一阶段的设计预留
Practice III

提问时帮 AI 定位文件。

14 · III
不好的提问 
"给我加个登录功能"

问题：AI 需要搜索 10+ 文件才能理解架构。每多一次搜索 = 多消耗 1-3K tokens + 增加误解风险。

→
好的提问 
在 src/auth/ 目录中实现登录。
参考 src/auth/register.ts 的结构。

约束：
- 使用现有的 validateEmail() 函数
- 返回值遵循 src/types/response.ts
  中的 AuthResponse 接口
- 添加测试到
  src/auth/__tests__/login.test.ts
提问模板 · Copy-ready 
[任务描述]
在 [文件位置] 实现 [具体需求]

[相关文件参考]
- 参考现有实现：[file A]
- 遵循约束：       [file B]
- 符合接口：       [file C]

[验收标准]
✓ 所有测试通过     ✓ TypeScript 无错误     ✓ 符合 CLAUDE.md
Practice IV

会话管理 — 维持 AI 的上下文质量。

15 · III

压缩 ≠ 丢一点细节；压缩 = 幻觉来源。

三个信号 · 立即重置
信号含义行动
压缩提示上下文 80%+清空
回复变水压缩幻觉新会话
反复追问失去上下文重述
一个会话的生命周期

五阶段管理

  1. 准备阶段
    粘贴 CLAUDE.md + 项目概览 · ~5K tokens
  2. 工作阶段
    3-5 个具体编程任务 · ~10-25K tokens
  3. 结束阶段
    总结改动、留下 TODO · ~2K tokens
  4. 清空 & 保存日志
    不直接清掉，先归档
  5. 新会话开启
    重新粘贴 CLAUDE.md，带新任务

借助 Prompt Cache：首次 10K，后续 0K。
